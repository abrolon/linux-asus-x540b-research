# Analysis of AMD Graphics Subsystem Instability on ASUS VivoBook X540B under Linux

This repository contains a practical case study documenting the investigation of graphics subsystem instability on legacy AMD hardware across several Linux distributions.

The main research text is written in Russian.  

## Repository Information

**Type:** Technical Case Study

**Status:** Completed

**Topic:** Linux / AMDGPU / Hardware Compatibility

**Objective:** Evaluate the feasibility of replacing Windows 10 with Linux on legacy AMD hardware while preserving GPU acceleration.  
---
## Анализ нестабильности графической подсистемы AMD на ASUS VivoBook X540B под Linux  

## Введение

Одной из моих целей было продлить срок полезного использования старого ноутбука ASUS VivoBook X540B за счёт перехода с Windows 10 на Linux.

Мотивация проекта основывалась на распространённом мнении о том, что современные Linux-дистрибутивы способны обеспечить более комфортную работу на устаревшем оборудовании благодаря меньшим требованиям к ресурсам системы.

Целью проекта была проверка данной гипотезы на реальном устройстве.

---

## Исходные данные

Устройство:

- ASUS VivoBook X540B
    
- AMD A6-9225 RADEON R4, 5 COMPUTE CORES 2C+3G
    
- 4 ГБ ОЗУ
## Конфигурация тестирования

### Операционные системы

- Linux Mint 22.3 XFCE
- Linux Mint 21.x (Kernel 5.15 LTS)
- MX Linux 25.2
- Windows 10 Pro

### Дополнительные инструменты

- Ventoy
- GRUB

> Точная версия Linux Mint 21.x не была зафиксирована в процессе тестирования.

Критерии успеха:

- стабильная работа системы;
    
- сохранение аппаратного ускорения графики;
    
- приемлемая производительность в повседневных задачах;
    
- улучшение пользовательского опыта по сравнению с Windows 10.

---

## Первичная проблема

Для тестирования был выбран Linux Mint 22.3 XFCE.

Проблемы появились ещё на этапе Live-системы:

- самопроизвольные перезагрузки;
    
- нестабильная работа графической подсистемы;
    
- невозможность завершить установку операционной системы.
    

Во время нескольких попыток установки ноутбук перезагружался на финальном этапе установки, вследствие чего система не устанавливалась корректно.

---

## Этап 1. Диагностика процесса установки

### Гипотеза №1

Проблема связана с режимом запуска Live-системы.

Действия:

- повторные попытки установки;
    
- использование Compatibility Mode.
    

Результат:

- установка завершилась успешно;
    
- операционная система была установлена на устройство.
    

Однако после завершения установки проблемы не исчезли и проявлялись уже во время обычной работы системы.

---

## Этап 2. Диагностика работы установленной системы

После успешной установки Linux Mint наблюдались:

- чёрные экраны;
    
- зависания графического интерфейса;
    
- самопроизвольные перезагрузки;
    
- нестабильная работа графической подсистемы.
    

Для проверки работоспособности использовался параметр:

```text
nomodeset
```

Результат:

- система запускалась значительно стабильнее;
    
- аппаратное ускорение графики фактически отключалось.
    

Данный вариант не соответствовал цели проекта, поскольку лишал устройство полноценного использования графического процессора.

---

### Гипотеза №2

Проблема связана с версией ядра Linux.

Действия:

- установка версии Linux Mint с более старым ядром семейства 5.x.
    

Результат:

- существенного улучшения не получено;
    
- проблемы сохранялись.
    

---

### Гипотеза №3

Проблема может быть связана с особенностями конкретного дистрибутива Linux.

Действия:

- переход на MX Linux.
    

Результат:

- система демонстрировала более предсказуемое поведение;
    
- однако проблема полностью не исчезла.
    

---

### Гипотеза №4

Проблема связана с работой графического стека и механизмами энергосбережения.

Действия:

В конфигурацию GRUB были добавлены параметры:

```text
amdgpu.dc=0
amdgpu.dpm=0
pcie_aspm=off
```

Результат:

- была достигнута наиболее стабильная конфигурация за всё время экспериментов;
    
- значительно сократилось количество зависаний и перезагрузок;
    
- система стала пригодной для повседневного использования.
    

При этом часть функциональности графической подсистемы была ограничена.

---

## Анализ результатов

Полученные результаты позволяют предположить наличие проблем совместимости между аппаратной платформой ноутбука и открытым графическим стеком Linux.

Наиболее стабильный результат был достигнут только на MX Linux после отключения ряда функций драйвера amdgpu и механизмов энергосбережения.

Несмотря на повышение стабильности, итоговая конфигурация сопровождалась ограничением возможностей графической подсистемы и снижением производительности относительно Windows 10 с проприетарным драйвером AMD.

---

## Ограничения исследования

- Не проводился анализ логов ядра и трассировок драйвера.
- Не выполнялась компиляция альтернативных версий ядра.
- Не тестировались другие версии Mesa.
- Выводы основаны на практических наблюдениях и результатах эксплуатации системы.

## Итог

Первоначальная цель проекта заключалась в том, чтобы повысить практическую ценность старого ноутбука посредством перехода на Linux.

В ходе исследования удалось добиться частичной стабилизации системы и получить рабочую конфигурацию на базе MX Linux.

Однако достигнутый результат не соответствовал критериям успеха проекта. Для обеспечения стабильности потребовалось ограничить функциональность графической подсистемы, а итоговая производительность и предсказуемость работы оказались ниже, чем при использовании Windows 10.

По результатам исследования было принято решение вернуть Windows 10 в качестве основной операционной системы для данного устройства.

## Практические результаты

В рамках проекта был получен практический опыт:

- диагностики проблем Linux на реальном оборудовании;
    
- работы с Live-системами и установщиками дистрибутивов;
    
- настройки параметров загрузки ядра;
    
- редактирования конфигурации GRUB;
    
- исследования поведения графической подсистемы AMD;
    
- проверки технических гипотез через последовательные эксперименты;
    
- документирования результатов технического исследования.
    
Основным выводом исследования стало понимание того, что успешность миграции на Linux определяется не только вычислительными возможностями оборудования, но и уровнем поддержки конкретной аппаратной платформы драйверами и графическим стеком операционной системы.
